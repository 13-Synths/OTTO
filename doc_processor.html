A DynArray that resizes to fit the RealTime bufferSize.

This is the container used in AudioProcessors, and it should be used in any place where the realtime data is copied out. It is resized on the bufferSizeChanged event

Struct otto::audio::ProcessData

template <int N>
struct ProcessData
{
    static constexpr int const channels = N;

    gsl::span<std::array<float, channels>> audio;

    gsl::span<midi::AnyMidiEvent> midi;

    long nframes;

    template <int outN = 0>
    ProcessData<outN> midi_only();

    ProcessData<N> audio_only();

    template <typename T>
    auto redirect(T& buf);

    ProcessData<N> slice(int idx, int length = -);

    auto begin();

    auto end();

    auto begin() const;

    auto end() const;
};

Non-owning package of data passed to audio processors

Function slice

ProcessData<N> slice(int idx, int length = -);

Get only a slice of the audio.

Requires:
parameter idx shall be in the range [0, nframes), and length shall be in range [0, nframes - idx]

Parameters

  • length
    — The number of frames to keep in the slice If length is negative, nframes - idx will be used
